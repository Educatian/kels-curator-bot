KELS Curator Bot 상세 가이드

KELS Curator Bot은 디스코드에 흩어지는 채용, CFP/RFP, 세미나, 학술 자료, 추천 논문, 이벤트 정보를 자동으로 모으고, 개인에게는 조용히 검색과 알림을 제공하며, 커뮤니티에는 필요한 순간에만 공개 큐레이션을 올리는 연구 커뮤니티 운영 도우미입니다.

Slash 응답은 기본 개인 공개 추천/리마인더는 지정 채널 공개 KELS + 좋아요 자동 반응 운영자 도구 포함

현재 문서는 멤버 안내, 운영자 안내, 자동화 흐름, 개인정보 범위, 문제 해결을 한 번에 볼 수 있도록 정리한 내부 가이드입니다.

개요 공개 범위 회원 명령어 공개 자동화 온보딩 운영/안전 채널 FAQ

한눈에 보는 역할

업데이트 기준: 2026년 6월 2일

14

등록된 slash command. 검색, digest, deadline, Q&A, profile, watch, 운영자 점검까지 포함합니다.

Private

일반 slash command 응답은 실행한 사람에게만 보입니다. 질문 내용이 전체 채널에 노출되지 않습니다.

Public

주간 추천 논문, Tech Signal, 이벤트 리마인더처럼 커뮤니티 전체에 유용한 내용만 지정 채널에 공개 게시됩니다.

Qwen

로컬 무료 LLM을 사용해 추천 논문, archive Q&A, CFP 분석, 온보딩 질문을 더 읽을 만하게 보강합니다.

S

Search

KELS 서버에 올라온 공개 글을 archive로 저장하고, 나중에 키워드나 질문으로 다시 찾을 수 있게 합니다.

C

Curation

OpenAlex, arXiv, GitHub 후보를 읽고 KELS 회원에게 의미 있는 추천 자료를 정리합니다.

O

Onboarding

자기소개 글에서 이름과 관심사를 읽고 개인 thread, 관련 원문, 참여 질문을 자동으로 제공합니다.

R

Reminder

공고와 이벤트 날짜를 추출해 마감 및 행사 리마인더를 제공합니다. 이미 끝난 이벤트는 제외합니다.

M

Moderation

명백한 스팸은 삭제하고, 역할 자동 태깅은 관리자/커뮤니케이션 계열 role을 건드리지 않습니다.

A

Ambient Support

새 포스팅과 thread 댓글에는 KELS 로고 이모지와 좋아요 이모지가 자동으로 달려 참여감을 높입니다.

무엇이 개인에게만 보이고, 무엇이 전체에 보이나요?

기능 보이는 사람 설명
/digest, /search, /deadlines 나에게만 실행한 사람에게만 결과가 보입니다. 채널에 질문이나 결과가 공개되지 않습니다.
/ask-kels, /cfp-helper, /topic-digest 나에게만 Qwen이 답변해도 기본 응답은 개인 공개입니다. 부끄러운 질문도 채널에 남지 않습니다.
/profile, /watch 나에게만 DM 가능 관심사와 watch 키워드는 개인 설정입니다. 관련 글이 올라오면 DM 알림을 받을 수 있습니다.
주간 추천 논문, KELS Tech Signal 전체 공개 #academic-resources에 커뮤니티 자료로 게시됩니다.
이벤트 1시간 전 리마인더 전체 공개 #announcement의 시간 포함 이벤트를 기준으로, 시작 1시간 전에 @everyone으로 알립니다.
자동 이모지 반응 전체 공개 인덱싱 대상 채널의 새 글과 thread 댓글에 KELS 로고 이모지와 좋아요 이모지를 답니다.
운영자 명령어 운영자 개인 응답 /health, /stats, /backfill 등은 운영자에게만 보이거나 운영자 권한이 필요합니다.

핵심 원칙

개인이 궁금해서 실행하는 명령은 개인에게만 보이고, 커뮤니티 전체가 같이 보면 좋은 큐레이션과 리마인더만 지정된 공개 채널에 올라갑니다.

회원용 slash command 상세 가이드

명령어를 입력하면 Discord가 옵션 입력 칸을 보여줍니다.

회원개인 응답

/help-kels

처음 사용할 때 가장 좋은 안내 명령입니다. 현재 사용할 수 있는 주요 기능과 예시를 간단히 보여줍니다.

언제 쓰나요?

  • 봇 기능을 처음 볼 때
  • 명령어 이름은 기억나는데 옵션이 헷갈릴 때
/help-kels
회원개인 응답

/digest

최근 올라온 KELS 게시물을 기간과 카테고리 기준으로 요약합니다. 채용, CFP, 세미나, 리소스를 빠르게 훑을 때 좋습니다.

추천 사용법

  • 일주일 동안 놓친 글 확인: category:all days:7
  • 최근 CFP만 확인: category:cfp days:30
  • 채용 공고 흐름 확인: category:jobs days:60
/digest category:all days:7
회원개인 응답

/search

KELS archive 안에서 키워드로 원문 글을 찾습니다. 특정 학회, 학교, 주제, 기회 이름을 찾을 때 가장 직접적입니다.

좋은 검색어 예시

  • learning analytics
  • AIED
  • assistant professor instructional design
  • NSF cyberlearning
/search query:"learning analytics" category:all
회원개인 응답Qwen

/ask-kels

검색보다 자연어 질문에 가깝습니다. archive에서 근거가 될 글을 찾고, Qwen이 관련 링크, 날짜, 채널, 근거를 묶어 답변합니다.

질문 예시

  • 최근 AIED 관련 CFP 뭐 있어?
  • learning sciences 교수 채용만 요약해줘.
  • AI ethics 관련해서 최근 서버에 올라온 자료 찾아줘.
  • 곧 마감되는 것 중 graduate student에게 맞는 기회가 있어?
/ask-kels query:"최근 AIED 관련 CFP 뭐 있어?"
회원개인 응답

/deadlines

게시글에서 추출한 마감일을 기준으로 다가오는 deadline을 보여줍니다. CFP, RFP, job posting, workshop 신청 마감 확인에 유용합니다.

주의할 점

  • 글에 날짜가 명확히 적혀 있어야 정확도가 높습니다.
  • 애매한 날짜 표현은 원문 링크를 열어 다시 확인하는 것이 좋습니다.
/deadlines days:60 category:all
회원개인 응답DM 알림

/watch

정확한 키워드를 등록하면, 새 글에 그 키워드가 포함될 때 DM 알림을 받을 수 있습니다. 짧고 구체적인 키워드에 적합합니다.

추천 키워드

  • AIED, CSCL, learning analytics
  • assistant professor, postdoc, special issue
/watch action:add keyword:"assistant professor"
회원개인 응답DM 알림

/profile

관심 주제를 저장합니다. 이후 관련 글이 올라오면 “왜 당신 관심사와 맞는지” 설명과 함께 DM을 받을 수 있고, 최근 KELS 원문도 추천됩니다.

추천 등록 방식

  • 너무 넓게: AI보다는 AI ethics in education
  • 연구 라인 중심: multimodal learning analytics
  • 기회 탐색 중심: instructional design faculty jobs
/profile action:add topic:"AI ethics in education"
회원개인 응답Qwen

/topic-digest

특정 주제에 대해 최근 archive를 묶어서 요약합니다. 단순 검색 결과보다 “흐름”을 보고 싶을 때 유용합니다.

사용 예시

  • AI ethics 관련 최근 논의
  • CSCL 관련 CFP/논문/자료 흐름
  • learning analytics 관련 채용과 리소스
/topic-digest topic:"AI ethics" days:30
회원개인 응답Field Explorer

/field-map

주제, 초록, CFP, 프로젝트 아이디어를 연결된 Field Explorer topic map 위에 놓아봅니다. 결과에는 가까운 topic 후보와 관련 KELS archive 원문이 함께 표시됩니다.

사용 예시

  • 내 연구 아이디어가 어느 분야 지점에 가까운지 볼 때
  • 추천 논문이나 CFP가 어떤 topic cluster와 연결되는지 볼 때
  • KELS archive에서 같은 방향의 원문 글을 같이 찾고 싶을 때
/field-map query:"multimodal learning analytics for teacher feedback" days:180
회원개인 응답Qwen

/cfp-helper

CFP/RFP 본문이나 링크를 넣으면 scope, 마감, 적합한 연구 주제, 준비 체크리스트를 정리합니다.

언제 쓰나요?

  • CFP가 내 연구와 맞는지 빠르게 판단하고 싶을 때
  • 공동저자에게 공유하기 전 핵심 정보를 정리하고 싶을 때
  • 마감과 제출물 요구사항을 놓치고 싶지 않을 때
/cfp-helper text:"CFP 본문 또는 링크"
회원개인 응답

포함하면 좋은 정보

  • 제목, 링크, 마감일
  • 분야 또는 키워드
  • KELS 회원에게 유용한 이유
/submit-cfp title:"..." url:"..." deadline:"..."

#announcement@everyone

이벤트 리마인더

#announcement에 올라온 이벤트에서 날짜, 시간, timezone, Zoom, RSVP, Google Form 링크를 감지합니다.

현재 동작

  • 시작 1시간 전 @everyone 리마인더
  • 이미 끝난 이벤트는 제외
  • 시간이 없는 공고는 리마인더 대상에서 제외
  • 이벤트 종료 후 자료/녹화/슬라이드 공유 thread 생성 가능
전체 공개KELS + 좋아요

자동 이모지 반응

인덱싱 대상 채널의 새 포스팅과 thread 댓글에는 KELS 로고 이모지와 좋아요 이모지가 자동으로 달립니다.

목적

  • 새 글이 방치된 느낌을 줄입니다.
  • 회원이 글을 올렸을 때 최소한의 반응을 즉시 제공합니다.
  • 스팸으로 삭제되는 메시지에는 반응하지 않습니다.
지정 채널마감 기반

마감 리마인더

CFP/RFP, 세미나, 신청형 이벤트의 날짜를 추출해 D-14, D-7, D-2 같은 시점에 다시 보여줄 수 있습니다.

운영 팁

  • 공고 글에 날짜와 timezone을 명확히 적으면 정확도가 올라갑니다.
  • 마감 링크와 제출 링크를 함께 넣으면 회원이 바로 행동하기 쉽습니다.

자기소개 기반 온보딩

새 회원이 “소개하고 끝”이 아니라 실제 참여로 이어지도록 돕는 흐름입니다.

자기소개 감지

#introduction에 올라온 공개 자기소개 글을 확인합니다.

이름과 관심사 추출

Discord 닉네임이 아니라 자기소개 본문에서 full name, 관심분야, 소속/단계, 찾는 정보를 읽습니다.

개인 thread 생성

thread 이름은 자기소개 글에서 추출한 Full Name 님 형식으로 만듭니다.

맞춤 참여 제안

관련 KELS 원문, 추천 채널, 첫 댓글 draft, 온보딩 질문, 1주 follow-up을 제공합니다.

새 회원에게 좋은 점

어디에 어떤 글을 올리면 좋을지, 어떤 자료부터 보면 좋을지, 누구와 대화가 이어질 수 있을지 빠르게 감을 잡을 수 있습니다.

운영자에게 좋은 점

환영 메시지를 매번 수동으로 쓰지 않아도, 새 회원의 관심사에 맞춘 구체적인 참여 경로가 자동으로 만들어집니다.

운영자 기능과 안전장치

운영자개인 응답

/health

봇이 현재 서버, 채널, archive, 권한 상태를 정상적으로 보고 있는지 확인합니다.

확인 항목

  • 봇 계정과 서버 이름
  • archive 게시글 수와 최신 인덱싱 시각
  • 현재 채널의 읽기, 쓰기, embed, reaction 권한
  • 인덱싱 대상 채널 설정
운영자개인 응답

/stats

archive에 저장된 글의 카테고리별, 채널별 분포를 확인합니다.

언제 쓰나요?

  • 어떤 채널이 가장 활발한지 볼 때
  • 자동 digest가 충분한 근거를 갖고 있는지 점검할 때
  • 커뮤니티 운영 리포트의 기초 통계를 볼 때
운영자개인 응답

/backfill

기존 채널의 최근 글을 수동으로 archive에 넣습니다. 봇을 새로 켰거나 특정 채널을 나중에 추가했을 때 유용합니다.

/backfill channel:#academic-resources limit:100
운영자공개 게시 가능

/post-digest

운영자가 의도적으로 digest를 공개 채널에 게시할 때 사용합니다. 일반 회원용 /digest와 다르게 공개 게시 목적입니다.

주의

공개 게시용이므로 채널 맥락에 맞는지 확인한 뒤 사용하는 것이 좋습니다.

운영 보조Qwen

Role 자동 태깅

회원의 공개 글을 바탕으로 관심분야 role을 추론합니다. 기존 role이 있으면 기존 role을 우선 사용하고, 필요하면 KELS: prefix를 붙인 새 role 후보를 만들 수 있습니다.

안전장치

  • Admin & Facilitator role은 자동 부여하지 않습니다.
  • CommunicationOfficer role은 자동 부여하지 않습니다.
  • admin, moderator, communication 계열 role은 자동 생성/부여에서 제외합니다.
  • confidence가 낮은 후보는 바로 부여하지 않고 운영자 검토 후보로 다루는 방향이 권장됩니다.
안전스팸 삭제

스팸 자동 정리

무자비하게 올라오는 명백한 스팸은 자동 삭제 대상입니다. 무료 Nitro, invite flood, 과도한 URL, 과도한 멘션, 반복 도배 등이 포함됩니다.

운영 원칙

  • 정상 회원의 긴 공고나 자료 공유를 스팸으로 오인하지 않도록 명백한 패턴 중심으로만 삭제합니다.
  • 삭제된 메시지는 archive에 저장하지 않고 자동 이모지 반응도 달지 않습니다.

자주 묻는 질문

슬래쉬 명령을 쓰면 전체 채널에 보이나요?

대부분의 회원용 slash command는 실행한 사람에게만 보입니다. 특히 검색, 질문, digest, deadline 확인은 개인 공개입니다.

봇이 내 DM으로 알림을 보내나요?

/watch나 /profile로 등록한 관심사와 맞는 글이 올라오면 DM 알림을 받을 수 있습니다. Discord DM 설정에 따라 수신이 제한될 수 있습니다.

왜 내 글에 KELS 이모지와 좋아요가 달렸나요?

새 포스팅이나 thread 댓글이 인덱싱 대상 채널에 올라오면 봇이 자동으로 기본 반응을 답니다. 환영과 참여 신호를 주기 위한 기능입니다.

자동 추천 논문은 아무 논문이나 고르나요?

아닙니다. 지정된 다섯 저널 안에서 후보를 고르고, 최신성, 공개 접근성, 인용 정보, KELS 맥락을 고려합니다.

Qwen 답변은 항상 맞나요?

Qwen은 archive 검색 결과를 읽기 쉽게 정리하는 보조 도구입니다. 중요한 마감, 제출 요건, 공식 정책은 원문 링크에서 다시 확인해야 합니다.

내 role이 자동으로 바뀔 수 있나요?

관심분야 role 추천/부여 기능은 운영 설정과 confidence 기준을 따릅니다. 관리자나 CommunicationOfficer 계열 role은 자동 부여되지 않습니다.

KELS Curator Bot 상세 가이드. 일반 회원용 slash command는 개인 응답이 기본이며, 공개 게시 자동화는 지정된 채널과 운영 목적에 맞춰 제한됩니다.
